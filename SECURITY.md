# Security · Безопасность

## EN

Report security issues privately. Do not open a public issue for vulnerabilities.

**What to include**

- Platform and app version
- Steps to reproduce
- Impact assessment if known

**What this repository hosts**

- Installers and APK release artifacts
- Release metadata and support docs

---

## RU

Сообщайте об уязвимостях приватно. Не создавайте публичный issue для security-проблем.

**Что указать**

- Платформа и версия приложения
- Шаги воспроизведения
- Оценку impact, если известна

**Что хранится в этом репозитории**

- Установщики и APK
- Метаданные релизов и документация поддержки
